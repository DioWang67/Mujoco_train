# 模擬結果判讀指南

給機構工程師、馬達工程師、專案成員閱讀。
不需要了解強化學習，只需要知道怎麼看數字。

---

## 怎麼產出這份數據

```cmd
python eval.py --log                    ← 讓 AI 走一集並開視窗，記錄每步數據
python eval.py --no-render --log        ← 不開視窗，只存資料（比較快）
python eval.py --episodes 3 --log       ← 跑 3 集，產出 eval_ep1.csv ~ eval_ep3.csv
python -m tools.plot_eval --save              ← 把 eval_ep1.csv 畫成圖，存成 PNG
python -m tools.plot_eval --file eval_ep2.csv --save  ← 指定要畫哪一集
```

產出的圖檔：`eval_ep1_plot.png`
產出的原始數據：`eval_ep1.csv`（可用 Excel 開啟）

## 圖表說明

### 面板 1：行走軌跡（俯視）

**看什麼：** 機器人走路的路徑從上方看下去。

| 觀察 | 代表意義 | 對機構的意義 |
|------|---------|------------|
| 路徑走直線 | 左右控制對稱 | 兩腿機構對稱性足夠 |
| 路徑偏轉或畫弧 | 左右不對稱 | 檢查兩腿長度/重量是否對稱 |
| Y 方向大幅飄移 | 側向不穩定 | 腳寬、髖關節位置可能需要調整 |

**判斷標準：**
- ✅ 走 10 秒內偏移 < 0.3m → 正常
- ⚠️ 偏移 0.3~0.8m → 輕微不對稱，可接受
- ❌ 偏移 > 0.8m 或持續旋轉 → 有結構性問題

---

### 面板 2：骨盆高度 / 姿態

**看什麼：** 機器人身體中心的高度，以及前後（Pitch）左右（Roll）的傾斜角度。

| 數值 | 正常範圍 | 超出代表 |
|------|---------|---------|
| 骨盆高度 | 1.00 ~ 1.10 m | 機器人在蹲著走或身體不穩 |
| Roll（左右傾） | ±5° 以內 | 側向穩定性不足，可能需要加寬腳距 |
| Pitch（前後傾） | ±8° 以內 | 重心前後位置不對，調整質量分佈 |

**對機構的意義：**
- 骨盆高度穩定 → 腿長設計合理，可以維持正常站姿
- Roll 大 → 腳距太窄，或髖關節側向剛性不足
- Pitch 大 → 重心不在腳掌正上方，前後質量分佈需調整

---

### 面板 3：前進速度

**看什麼：** 機器人實際走路速度（藍線）vs 目標速度（虛線 1.0 m/s）。

| 觀察 | 代表意義 |
|------|---------|
| 藍線接近虛線 | 控制精準，機構阻力合理 |
| 藍線明顯低於目標 | 摩擦力過大，或馬達扭矩不足 |
| 速度劇烈抖動 | 步態不穩，需要更多訓練或調整機構 |
| vel_y（黃線）幅度大 | 側滑嚴重，腳底摩擦力不足或腳型問題 |

---

### 面板 4 & 5：左腿 / 右腿關節角度

**看什麼：** 走路過程中每個關節的角度變化（單位：度）。

**各關節說明：**

| 關節 | 功能 | 走路時正常範圍 |
|------|------|-------------|
| hip_pitch（髖關節前後）| 腿的前後擺動 | -30° ~ +20° |
| hip_roll（髖關節左右） | 側向平衡調整 | -10° ~ +10° |
| hip_yaw（髖關節旋轉） | 腳尖方向 | -8° ~ +8° |
| knee（膝關節）| 支撐與緩衝 | 0° ~ +65° |
| ankle（踝關節）| 推進與落地緩衝 | -25° ~ +10° |

**怎麼用這個數字：**

機構設計時，關節的**物理行程限制**必須大於模擬用到的範圍，建議預留 **20% 餘量**。

範例：模擬中膝關節最大 58°，則機構設計至少需要 **70°** 的行程。

**左右對比：**
- 左右腿的曲線形狀應該相似，只差半個週期（相位差 180°）
- 如果兩腿曲線差異很大 → 機器人走路不對稱，可能是機構或控制問題

---

### 面板 6：各關節最大扭矩

**看什麼：** 整個走路過程中，每個關節承受過的最大扭矩（Nm）。

**這個數字直接影響馬達選型。**

| 關節 | 模擬最大扭矩 | 建議馬達額定扭矩 |
|------|------------|---------------|
| knee | （看圖）| 模擬值 × 1.5 以上 |
| hip_pitch | （看圖）| 模擬值 × 1.5 以上 |
| ankle | （看圖）| 模擬值 × 1.3 以上 |
| hip_roll | （看圖）| 模擬值 × 1.3 以上 |

> **為什麼要乘以 1.5？**
> 模擬的地面是平整的，真實地面有坡度、不平整、突發擾動，扭矩需求會更高。
> 加速、緊急停止等動態情況也會產生更大的瞬間扭矩。

## CSV 原始數據欄位說明

用 Excel 開啟 `eval_ep1.csv`，各欄位說明：

| 欄位名稱 | 單位 | 說明 |
|---------|------|------|
| step | - | 時間步（1 step = 0.02 秒） |
| pelvis_x/y/z | m | 骨盆（身體中心）的 XYZ 座標 |
| roll_deg | ° | 身體左右傾斜角 |
| pitch_deg | ° | 身體前後傾斜角 |
| yaw_rate | rad/s | 身體旋轉速率 |
| vel_x | m/s | 前進速度 |
| vel_y | m/s | 側向速度（理想為 0） |
| vel_z | m/s | 垂直速度（理想接近 0） |
| `{關節名}_pos_deg` | ° | 該關節目前角度 |
| `{關節名}_vel_dps` | °/s | 該關節轉動角速度 |
| `{關節名}_torque` | Nm | 該關節目前施加扭矩 |

## 常見問題 QA

**Q：模擬的扭矩數字可以直接拿來選馬達嗎？**

可以當參考，但要乘上安全係數（建議 1.5x）。
模擬的物理是理想化的，真實馬達還有效率損耗、慣性、溫度影響。

**Q：模擬走路很好，真機一定也走得好嗎？**

不一定。模擬和真實之間有「Sim-to-Real Gap」，主要差異在：
- 地面摩擦力不同
- 馬達有延遲和噪訊
- 零件有彈性形變

需要加做 Domain Randomization 訓練才能縮小這個差距。

**Q：模擬走路速度 1.0 m/s，真機也能走這麼快嗎？**

模擬速度是設定值，真機能否達到取決於馬達功率和腿部慣量。
建議先以 0.5 m/s 為真機初始目標，確認穩定後再提速。

**Q：怎麼知道模擬夠不夠準確？**

目前這個模擬適合用於：
- ✅ 關節角度範圍設計參考
- ✅ 相對扭矩大小的比較（哪個關節負擔最重）
- ✅ 步態週期和節奏的參考
- ❌ 精確的扭矩數值（需要驗證）
- ❌ 結構應力分析（需要 FEA）
